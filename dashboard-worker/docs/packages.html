⚠️ Known Limitations & Edge Cases

Important limitations to be aware of when using bun pm pkg:

# ❌ Version Numbers with Dots (Common Issue)
# This will FAIL:
bun pm pkg get metadata.roadmap.v2.2.0        # InvalidPath error
bun pm pkg get "metadata.roadmap[\"v2.2.0\"]" # InvalidPath error

# ✅ Workaround - Access the full object:
bun pm pkg get metadata.roadmap                # Returns complete roadmap

# ✅ Alternative - Restructure data to avoid dots:
# Use underscores or hyphens instead of dots in version keys
"roadmap": {
  "v2_2_0": "Enhanced monitoring and alerting",
  "v2_3_0": "Real-time notifications"
}
Why This Happens: The bun pm pkg system has trouble parsing paths containing dots, even with bracket notation. This affects version numbers, which are common in package.json files.
Best Practice: Avoid dots in property names when possible. Use underscores, hyphens, or camelCase for version identifiers.
