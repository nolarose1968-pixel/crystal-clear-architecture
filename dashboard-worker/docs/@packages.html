‚ö†Ô∏è Known Limitations & Edge Cases

Important limitations discovered during testing:

üö® Array Index Manipulation

Critical Limitation: Setting array indices doesn't work like JavaScript arrays. It overwrites the entire array instead of extending it.

# ‚ùå This will FAIL - overwrites entire array
bun pm pkg set "config.envValidation.required[4]"="NEW_VAR"

# ‚úÖ This works - sets entire array
bun pm pkg set "config.envValidation.required"='["VAR1", "VAR2", "VAR3", "VAR4", "NEW_VAR"]'

üö® Version Numbers with Dots

Common Issue: The bun pm pkg system has trouble parsing paths containing dots, even with bracket notation.

# ‚ùå This will FAIL
bun pm pkg get metadata.roadmap.v2.2.0
bun pm pkg get "metadata.roadmap[\"v2.2.0\"]"

# ‚úÖ Workaround - access the full object
bun pm pkg get metadata.roadmap

# ‚úÖ Alternative - restructure data
"roadmap": {
  "v2_2_0": "Enhanced monitoring",
  "v2_3_0": "Real-time notifications"
}
üí° Best Practice: Avoid dots in property names when possible. Use underscores, hyphens, or camelCase for version identifiers.

üìã Dot Notation Examples

# Basic Properties

üîó Bracket Notation Examples

# Array Access
bun pm pkg get "contributors[0]"        # First contributor object
bun pm pkg get "contributors[1].role"   # Second contributor role
bun pm pkg get "scripts[0]"             # First script

# Special Characters (colons, hyphens, etc.)
bun pm pkg get 'scripts["test:watch"]'   # Get test:watch script
bun pm pkg get 'scripts["env:audit"]'    # Get env:audit script
bun pm pkg get 'scripts["deps:audit"]'   # Get deps:audit script

# Complex Property Names

bun pm pkg get "metadata.environment.supportedEnvironments[2]" # Third environment

‚ö° Mixed Notation Patterns

# Combine dot and bracket notation

bun pm pkg get "scripts[\"test:watch\"]" # Bracket with quotes
bun pm pkg get config.database.pool.max # Pure dot notation

# Setting Values with Mixed Notation
bun pm pkg set config.environment="staging" # Dot notation
bun pm pkg set "scripts[\"deploy:canary\"]"="bun run deploy:staging" # Bracket notation
bun pm pkg set "config.envValidation.required[4]"="NEW_VAR" # Mixed notation
üí° Pro Tip: Use dot notation for simple nested properties and bracket notation for array access or special characters. Both work consistently across all bun pm pkg commands!
