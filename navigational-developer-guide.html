❓ Frequently Asked Questions

Where do I add a new customer feature?
Navigate to: src/components/customer/
Add your new functionality following the existing patterns. Update types in src/components/customer/core/customer-interface-types.ts and implement in the appropriate module (search, forms, etc.).
How do I add a new API endpoint?
Navigate to: src/api/controllers/
Choose the appropriate domain controller (settlement, adjustment, balance, etc.) or create a new one. Follow the established pattern with proper error handling and TypeScript types.
Where do I add language translations?
Navigate to: src/i18n/
Add your L-XXXX key to fire22-language-keys.json with translations in all 4 supported languages (English, Spanish, Portuguese, French). Update the validation rules if needed.
How do I add a new Telegram command?
Navigate to: src/telegram/commands/
Create or modify command files, then register the language key in src/telegram/telegram-constants.ts. Update the multilingual bot integration in src/telegram/multilingual-telegram-bot.ts.
Where do I add financial/balance logic?
Navigate to: src/finance/
Choose the appropriate module: validation for rules, audit for logging, notifications for alerts, or the main finance module for core operations. Update shared types in core/types/finance/.
How do I add agent management features?
Navigate to: src/hierarchy/
Use the agent manager for profile operations, commission manager for payout logic, or permissions module for access control. Update hierarchy types in core/types/hierarchy/.
💡 Pro Tip: Always check the existing patterns in each module before adding new code. The codebase follows consistent architectural patterns that should be maintained for long-term maintainability.
